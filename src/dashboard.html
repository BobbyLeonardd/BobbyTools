Panduan Lengkap (Biar Lo Ga Keder)

Baca bentar doang, janji. Gue nulis ini karena gue juga males jawabin pertanyaan yang sama berulang-ulang. Abis baca ini lo bakal ngerti semua isi BobbyTools.

Dua mode, pilih salah satu (atau dua-duanya)

BobbyTools punya dua cara pakai. Mode Web/Router (halaman ini) buat yang mau anti-limit 429 otomatis. Mode Klasik/CLI buat yang cuma mau launch cepat satu akun tanpa ribet. Isi provider & akun sekali, dua mode langsung kepake dua-duanya — datanya sama (~/.bobbytools/config.json).

0. Install dulu (kalo belum) 📦

BobbyTools jalan di atas Node.js — minimal versi 18. Cek dulu punya lo: node -v. Kalo belum ada atau ketuaan, ambil di nodejs.org (pilih yang LTS).

Abis itu install global lewat npm — sekali doang, langsung bisa dipanggil bobby dari terminal mana aja:

npm install -g bobbytools

Cek berhasil: ketik bobby -v, harusnya keluar nomor versinya. Kalo shell-nya bilang "command not found", biasanya folder global npm belum masuk PATH — restart terminal dulu, atau cek npm bin -g.

Mau update ke versi terbaru nanti? Gampang, tinggal bobby update — dia ngecek npm dan nawarin update sendiri.

1. Mode Web (Router) — yang bikin alat ini worth it 🌟

Bobby jalan di background jadi "calo" (middleman) di 127.0.0.1:13337. CLI ngoding lo nembak ke Bobby, Bobby yang nyuntikin API key asli dan muterin akun kalo kena limit. Terminal lo gak pernah tau ada drama di belakang.

Langkah 1 — Daftarin Provider & Akun:

  • Klik Add Provider (menu Providers). Ada dua cara: pilih From Template (Groq, OpenAI, Gemini, dll udah jadi) atau Custom Setup kalo provider lo aneh.
  • Buka provider itu, klik Add Account. Masukin semua API key yang lo punya — akun A sampe Z. Makin banyak akun, makin lama lo baru kena limit.

Langkah 2 — Sambungin CLI ngoding lo (Aider / Opencode / Cursor):

Lo cuma perlu ngibulin CLI biar dia ngira Bobby itu server aslinya. Buka terminal, copas sesuai OS lo:

Mac/Linux/GitBash : export OPENAI_BASE_URL="http://127.0.0.1:13337/v1"
Windows PowerShell: $env:OPENAI_BASE_URL="http://127.0.0.1:13337/v1"
Windows CMD       : set OPENAI_BASE_URL=http://127.0.0.1:13337/v1

(Gak usah repot ngeset OPENAI_API_KEY — isi sk-bobby atau asdf juga bodo amat. Yang nyuntikin key asli itu Bobby, bukan lo.)

Pake CLI yang maunya standar Anthropic (kayak claude-code)? Ganti prefix env var-nya, URL-nya sama: ANTHROPIC_BASE_URL + ANTHROPIC_API_KEY.

Langkah 3 — Jalanin, pake format provider/model:

Nama depan itu yang bikin router tau mau nembak ke provider mana.

opencode -m groq/llama3-70b-8192
aider --model openrouter/anthropic/claude-3-haiku

Bingung nama provider-nya apa? Liat aja di list Providers sebelah — nama yang lo kasih itu yang dipake (spasi jadi strip, misal "My Router" → my-router).

✨ Yang terjadi di belakang layar:

  • Auto-Rotate + Round-Robin: Punya 5 akun Groq? Traffic dibagi rata ke lima-limanya. Begitu satu kena 429 (atau 401/402), Bobby diem-diem pindah ke akun berikutnya dan nge-retry. Terminal lo taunya cuma sukses.
  • Cooldown otomatis: Akun yang kena limit ditandai "limited", tapi balik "active" sendiri setelah cooldown lewat (default ~60 detik). Jadi lo gak perlu ngungkit-ungkit akun manual tiap limit ilang.
  • Universal Fallback lintas-provider: Kalo SEMUA akun Groq abis, Bobby nyari provider lain yang punya model persis sama (misal OpenRouter yang juga punya llama3-70b) dan pindah ke situ mid-request. Syaratnya: model itu kedaftar di provider cadangannya.

2. Mode Klasik (CLI Launcher) — buat yang paling males 💤

Males buka browser, males ngeset env var, cuma mau jalanin satu akun? Tutup web ini, buka terminal, ketik bobby.

  • Pilih Start Session → klik-klik: Provider → Akun → Model.
  • Bobby launch target CLI lo (misal opencode) dengan API key udah kesuntik di memori proses. Gak ada acara ngetik export manual.
  • Pro tip: besoknya tinggal bobby go — langsung buka sesi terakhir lo, tanpa klik-klik lagi.

3. Ngatur Model tiap Provider 🧠

Buka provider → Edit Provider → Edit Models. Di situ lo bisa CRUD daftar model: tambah manual, list, rename, hapus.

  • Punya endpoint /models? Ada tombol Fetch/Refresh buat narik daftar model otomatis dari API-nya. Hasilnya digabung, gak nimpa yang udah ada.
  • Gak punya endpoint? Santai — tinggal ketik nama model manual. Sama aja.

⚠️ Soal provider yang base URL-nya lokal

Kalo lo bikin provider yang nunjuk ke alamat lokal (localhost, 127.0.0.1, 0.0.0.0, ::1) — misal nunjuk balik ke router Bobby sendiri, atau ke Ollama/LM Studio — endpoint model-nya sengaja dimatiin. Bobby gak bakal narik model dari situ, biar gak ada acara serap-menyerap muter-muter yang bikin daftar model jadi sampah. Buat provider lokal: isi model manual lewat Edit Models. Itu wajar, bukan bug.

📚 Daftar Perintah (Commands) BobbyTools

  • bobby
    Buka menu utama interaktif (nambah akun, pilih model, atur provider, dll).
  • bobby go
    Jalan pintas orang malas. Langsung buka sesi terakhir lo tanpa klik-klik menu.
  • bobby serve
    Nyalain router + Web Dashboard di depan layar (foreground). Terminal ditutup, server ikut mati.
  • bobby serve-bg
    Nyalain di background (daemon). Terminal bebas ditutup, server tetep idup, browser dibukain otomatis.
  • bobby list
    Nampilin semua provider & akun yang tersimpan, tanpa masuk menu.
  • bobby update
    Ngecek versi terbaru di npm. Kalo ada yang baru, langsung ditawarin update otomatis (tinggal Enter). Kalo udah paling baru, dia bilang.
  • bobby -v / --version
    Cek versi yang lagi kepasang.
  • bobby -h / --help
    Nampilin contekan bantuan.

🔧 Kalo Mampet (Troubleshooting)

  • 401 Unauthorized terus? API key di akunnya salah/expired, atau format model-nya kebalik. Cek pake tombol Test di menu akun (mode CLI).
  • "Provider not found"? Prefix model lo gak cocok sama nama provider. Samain dengan nama di list (spasi → strip).
  • Model lokal gak muncul? Emang gak diserep otomatis (baca kotak kuning di atas). Tambah manual di Edit Models.
  • Semua kacau, mau mulai bersih? Menu Settings → Factory Reset (mode CLI). Config lama ada backup-nya di ~/.bobbytools/config.backup.json.

🗑️ Cara Uninstall (Kalo Udah Bosen)

Matiin dulu router yang lagi jalan (tombol Shutdown di pojok, atau bobby → Stop Web Dashboard). Terus:

npm uninstall -g bobbytools

Itu ngehapus program-nya doang. Config lo (semua API key & provider) masih nyangkut di ~/.bobbytools/. Kalo mau hapus tuntas sekalian datanya:

Mac/Linux         : rm -rf ~/.bobbytools
Windows PowerShell: Remove-Item -Recurse -Force $HOME\.bobbytools

Hati-hati — langkah kedua ngehapus semua API key yang lo simpen, gak ada undo. Kalo cuma mau install ulang, skip langkah ini biar config lo kejaga.
